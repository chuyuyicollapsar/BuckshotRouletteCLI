# Buckshot Roulette 部署指南

本文档描述项目的部署设计、当前已实现的部署产物，以及服务端 Docker 镜像的构建、分发和运行方式。

## 部署对象

项目运行时分为两个对象：

- 服务端：提供 HTTP API、WebSocket 事件推送、房间管理、游戏状态和 AI 回合调度。
- CLI 客户端：玩家在终端中运行，用于创建房间、加入房间、提交行动和接收事件。

当前仓库已经提供两个 Python 命令入口：

```bash
buckshot-server
buckshot-client
```

当前已实现的部署形态：

- 服务端可以构建为 Docker 镜像并作为容器运行。
- CLI 客户端可以通过 Python 包安装后运行。
- CLI 客户端可以打包成 Windows 和 Linux 独立可执行文件。

## 推荐分发模型

当前推荐的部署模型是：

```text
开发者/发布者
  构建服务端 Docker 镜像
  发布镜像到镜像仓库，或导出为 tar 文件

服主/服务器管理员
  获取服务端镜像
  使用 docker run 或 docker compose 启动服务端

玩家
  安装或获取 CLI 客户端
  使用 CLI 客户端连接服务端地址
```

服务端 Docker 镜像解决的是服务器部署问题。它不包含玩家本机的独立 CLI 程序。玩家客户端的独立分发需要单独的打包流程。

## Docker 相关文件

仓库提供以下 Docker 部署文件：

```text
Dockerfile
docker-compose.yml
.dockerignore
```

`Dockerfile` 定义服务端镜像如何构建。

`docker-compose.yml` 定义本地或单机服务器如何启动服务端容器。

`.dockerignore` 控制 Docker 构建上下文，避免把虚拟环境、缓存、构建产物和测试目录复制进镜像构建过程。

## 镜像和容器

Docker 部署中需要区分两个概念：

- 镜像 image：可分发、可复用的服务端运行包。
- 容器 container：由镜像启动出来的服务端运行实例。

构建过程：

```text
项目源码 + Dockerfile
  -> docker build
  -> 服务端镜像
```

运行过程：

```text
服务端镜像
  -> docker run / docker compose up
  -> 服务端容器
```

镜像由 Docker 管理，不存放在项目工作区。可以通过以下命令查看：

```bash
docker images
```

容器也由 Docker 管理，可以通过以下命令查看：

```bash
docker ps
docker ps -a
```

如果需要把镜像变成普通文件进行离线分发，可以导出为 tar 文件：

```bash
docker save -o buckshot-roulette-server_0.1.0.tar buckshot-roulette-server:0.1.0
```

另一台机器可以导入：

```bash
docker load -i buckshot-roulette-server_0.1.0.tar
```

## 构建服务端镜像

在项目根目录执行：

```bash
docker build -t buckshot-roulette-server:0.1.0 .
```

构建完成后查看镜像：

```bash
docker images buckshot-roulette-server
```

如果使用相同 tag 重新构建：

```bash
docker build -t buckshot-roulette-server:0.1.0 .
```

新的镜像会接管 `buckshot-roulette-server:0.1.0` 这个 tag。旧镜像层可能仍保留在 Docker 内部存储中，可以按需清理：

```bash
docker image prune
```

## 本地运行服务端容器

前台运行：

```bash
docker run --rm \
  --name buckshot-roulette-server \
  -p 8000:8000 \
  -v buckshot-data:/data \
  buckshot-roulette-server:0.1.0
```

后台运行：

```bash
docker run -d \
  --name buckshot-roulette-server \
  --restart unless-stopped \
  -p 8000:8000 \
  -v buckshot-data:/data \
  buckshot-roulette-server:0.1.0
```

查看日志：

```bash
docker logs -f buckshot-roulette-server
```

停止容器：

```bash
docker stop buckshot-roulette-server
```

删除已停止的容器：

```bash
docker rm buckshot-roulette-server
```

验证服务：

```bash
curl http://127.0.0.1:8000/health
```

预期返回：

```json
{"status":"ok"}
```

## 使用 Docker Compose

Docker Compose 适合本地开发、单机部署或需要固定配置的服务器部署。

构建并启动：

```bash
docker compose up -d --build
```

查看服务状态：

```bash
docker compose ps
```

查看日志：

```bash
docker compose logs -f
```

停止并删除容器：

```bash
docker compose down
```

停止并删除容器，同时删除命名数据卷：

```bash
docker compose down -v
```

`docker-compose.yml` 默认行为：

- 构建镜像：`buckshot-roulette-server:0.1.0`
- 启动容器：`buckshot-roulette-server`
- 暴露端口：宿主机 `8000` -> 容器 `8000`
- 挂载数据卷：`buckshot-data:/data`
- 设置环境变量：`BUCKSHOT_DATA_DIR=/data`
- 容器异常退出后自动重启：`restart: unless-stopped`

## 发布服务端镜像

服务端镜像可以通过两种方式分发。

### 发布到镜像仓库

给镜像打远程仓库 tag：

```bash
docker tag buckshot-roulette-server:0.1.0 yourname/buckshot-roulette-server:0.1.0
```

推送：

```bash
docker push yourname/buckshot-roulette-server:0.1.0
```

服主获取并运行：

```bash
docker pull yourname/buckshot-roulette-server:0.1.0
docker run -d \
  --name buckshot-roulette-server \
  --restart unless-stopped \
  -p 8000:8000 \
  -v buckshot-data:/data \
  yourname/buckshot-roulette-server:0.1.0
```

### 离线 tar 文件分发

导出镜像：

```bash
docker save -o buckshot-roulette-server_0.1.0.tar buckshot-roulette-server:0.1.0
```

服主导入：

```bash
docker load -i buckshot-roulette-server_0.1.0.tar
```

服主运行：

```bash
docker run -d \
  --name buckshot-roulette-server \
  --restart unless-stopped \
  -p 8000:8000 \
  -v buckshot-data:/data \
  buckshot-roulette-server:0.1.0
```

## Docker Desktop 使用方式

Docker Desktop 可以管理镜像、容器、日志、端口和数据卷。

常见使用方式：

- 使用命令行 `docker build` 构建镜像，然后在 Docker Desktop 的 Images 中查看。
- 使用命令行 `docker run` 或 `docker compose up` 启动容器，然后在 Docker Desktop 的 Containers 中查看。
- 使用 `docker load -i xxx.tar` 导入离线镜像，然后在 Docker Desktop 中运行。

Docker Desktop 不是直接运行项目源码目录。项目源码只在构建镜像时使用。真正运行的是镜像启动出来的容器。

## 客户端连接服务端

服务端启动后，客户端连接地址格式为：

```text
http://服务器地址:8000
```

本机连接：

```bash
buckshot-client --server http://127.0.0.1:8000 --name Alice
```

局域网连接：

```bash
buckshot-client --server http://192.168.1.10:8000 --name Alice
```

公网连接：

```bash
buckshot-client --server http://example.com:8000 --name Alice
```

如果使用独立客户端程序，将上面的 `buckshot-client` 替换为对应平台的可执行文件路径。

Windows：

```powershell
.\buckshot-client.exe --server http://127.0.0.1:8000 --name Alice
```

Linux：

```bash
chmod +x ./buckshot-client
./buckshot-client --server http://127.0.0.1:8000 --name Alice
```

## 客户端独立程序

仓库提供 PyInstaller 构建脚本，用于生成独立 CLI 客户端。

Windows 构建脚本：

```text
scripts/build_client_windows.ps1
```

Linux 构建脚本：

```text
scripts/build_client_linux.ps1
scripts/build_client_linux.sh
```

PyInstaller 入口：

```text
scripts/pyinstaller/buckshot_client_entry.py
```

Linux 客户端通过 Docker 构建，使用：

```text
Dockerfile.client-linux
```

### 构建 Windows 客户端

在 Windows 项目根目录执行：

```powershell
.\scripts\build_client_windows.ps1
```

输出文件：

```text
dist/client/windows/buckshot-client.exe
```

验证：

```powershell
.\dist\client\windows\buckshot-client.exe --help
```

### 构建 Linux 客户端

在 Windows PowerShell 中执行：

```powershell
.\scripts\build_client_linux.ps1
```

或在 Linux / WSL shell 中执行：

```bash
bash scripts/build_client_linux.sh
```

输出文件：

```text
dist/client/linux/buckshot-client
```

验证：

```bash
./dist/client/linux/buckshot-client --help
```

如果在 Windows 上验证 Linux 可执行文件，可以使用临时 Linux 容器：

```powershell
docker run --rm -v "${PWD}\dist\client\linux:/client" debian:trixie-slim /client/buckshot-client --help
```

### 客户端构建产物

客户端独立程序是普通文件，可以直接分发给玩家：

```text
dist/client/windows/buckshot-client.exe
dist/client/linux/buckshot-client
```

这些产物位于 `dist/` 目录。`dist/` 是构建输出目录，不建议提交到 Git；发布版本时应把对应文件上传到 Release 或分发包中。

### 客户端构建限制

PyInstaller 通常不能可靠地跨平台打包：

- Windows 客户端应在 Windows 环境构建。
- Linux 客户端应在 Linux 环境构建。

本仓库的 Linux 客户端脚本使用 Docker 构建 Linux ELF 文件，因此可以从 Windows 主机触发 Linux 客户端构建。

当前未提供 macOS 客户端构建脚本。macOS 客户端应在 macOS 环境中单独构建。

## Python 包方式安装客户端

构建 wheel：

```bash
python -m pip wheel . --no-deps -w dist
```

安装：

```bash
python -m pip install dist/buckshot_roulette_cli-0.1.0-py3-none-any.whl
```

运行客户端：

```bash
buckshot-client --server http://127.0.0.1:8000 --name Alice
```

当前项目要求：

```toml
requires-python = ">=3.14"
```

因此使用 Python 包方式运行客户端时，目标机器也需要 Python 3.14 或更高版本。

## 数据和配置

服务端当前使用内存存储保存房间和对局状态：

- 容器重启后，房间会消失。
- 容器重启后，正在进行的对局会丢失。
- 这是当前 MVP 后端设计的一部分。

LLM / AI 配置使用文件持久化：

```text
/data/llm_config.json
```

Docker 部署中默认设置：

```bash
BUCKSHOT_DATA_DIR=/data
```

使用 `docker run` 时建议挂载命名卷：

```bash
-v buckshot-data:/data
```

使用 Docker Compose 时，`docker-compose.yml` 已经配置：

```yaml
volumes:
  - buckshot-data:/data
```

## LLM provider 依赖

默认服务端镜像只安装基础依赖：

- fastapi
- uvicorn[standard]

默认镜像可以使用内置的 `fake_local` AI 预设，不需要外部 LLM provider。

如果需要真实 LLM provider，可以构建包含所有可选 LLM 依赖的镜像：

```bash
docker build --build-arg EXTRAS=llm -t buckshot-roulette-server:0.1.0-llm .
```

运行时通过环境变量注入 API Key：

```bash
docker run -d \
  --name buckshot-roulette-server \
  --restart unless-stopped \
  -p 8000:8000 \
  -v buckshot-data:/data \
  -e OPENAI_API_KEY=your_api_key \
  buckshot-roulette-server:0.1.0-llm
```

推荐在 LLM provider 配置中使用 `api_key_env` 引用环境变量。不要把真实 API Key 明文写入 `llm_config.json`。

## 端口和网络

容器内监听地址：

```text
0.0.0.0:8000
```

宿主机端口映射：

```text
8000:8000
```

本机访问：

```text
http://127.0.0.1:8000
```

局域网访问：

```text
http://服务器局域网IP:8000
```

如果部署到公网服务器，需要处理：

- 防火墙或安全组放行端口。
- 域名解析。
- TLS 证书。
- 反向代理。

如果使用 Nginx、Caddy 或其他反向代理，需要确保 WebSocket Upgrade 正常转发，否则 CLI 无法稳定接收房间事件。

## 升级流程

如果服务端代码发生变化，需要重新构建服务端镜像：

```bash
docker build -t buckshot-roulette-server:0.1.1 .
```

停止旧容器：

```bash
docker stop buckshot-roulette-server
docker rm buckshot-roulette-server
```

使用新镜像启动：

```bash
docker run -d \
  --name buckshot-roulette-server \
  --restart unless-stopped \
  -p 8000:8000 \
  -v buckshot-data:/data \
  buckshot-roulette-server:0.1.1
```

如果使用 Docker Compose，更新镜像版本或源码后执行：

```bash
docker compose up -d --build
```

如果客户端代码发生变化，需要重新发布客户端安装包。当前阶段是重新构建 wheel 并分发；未来如果加入独立客户端程序，则需要重新打包客户端可执行文件。

## WSL Ubuntu

如果在 Windows 上使用 WSL Ubuntu，并且 Docker Desktop 已启用 WSL integration，可以在 WSL 中进入项目目录：

```bash
cd /mnt/c/swnfv/code/BuckshotRouletteCLI
```

构建并启动服务端：

```bash
docker compose up -d --build
```

验证：

```bash
curl http://127.0.0.1:8000/health
```

注意：Docker 方式运行服务端不需要在 WSL 中安装项目 Python 依赖。只有当你要在 WSL 中运行 Python 包形式的 `buckshot-client` 时，才需要安装 Python 3.14 和项目 wheel。
