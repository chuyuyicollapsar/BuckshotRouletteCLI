# 恶魔轮盘游戏引擎设计

---

## 一、游戏概述

恶魔轮盘多人模式的核心规则与游戏引擎设计。玩家轮流用一把霰弹枪对准自己或他人射击，子弹有 LIVE（实弹）和 BLANK（空包）两种。每局 3 场比赛，每场比赛存活到最后的人获胜。

本项目只按多人模式设计，不处理单人/故事模式的流程差异。

支持 **2~4 名玩家** 对战。

---

## 二、游戏结构

### 2.1 一场比赛（Match）

一局游戏共 **3 场比赛（三场）**。

每场比赛流程：
1. 所有玩家根据对局配置获得初始血量（引擎默认策略：从 {3, 4, 5} 中随机取一个值）
2. 使用道具和子弹对决
3. 直到只剩 **一人存活**（其他人 HP ≤ 0）时该场比赛结束
4. 下一场比赛：**道具清零、血量重新随机**，一切从头开始

> 3 场全打满，即使一方已连胜 2 场。

### 2.2 单场比赛的轮次（Reload Round）

一场比赛内，子弹是分批装入的。每批子弹称为 **一轮（Reload Round）**：

1. 装弹：根据对局配置生成若干发子弹（引擎默认策略：2~8 发），展示 LIVE/BLANK 数量分布，随后随机排列装入弹匣
2. 发道具：所有存活玩家获得新道具
3. 轮流行动：按回合规则进行
4. 子弹打空 → 进入下一轮（重新装弹 + 发道具）
5. **只剩一人存活 → 比赛立即结束**

### 2.3 胜利条件

| 条件 | 结果 |
|------|------|
| 比赛内只剩一人存活 | 该玩家赢得该场比赛 |
| 3 场全打完，赢更多场的一方 | 赢得整局游戏 |

### 2.4 回合顺序

按照 **当前回合方向** 在玩家编号中循环。默认正向（0→1→2→3→0→1...），Remote 会反转方向。

"回合"与"轮次"无关，只与当前行动权有关：
- 每个行动权称为一个**回合（Turn）**
- 一轮（Reload Round）中包含多个回合，直到子弹打空
- 切换回合时跳过死亡玩家和被干扰器跳过的玩家

## 三、弹匣规则

### 3.1 子弹数量

每轮子弹由对局配置决定。

引擎默认策略：随机生成 **2~8 发** 子弹。

### 3.2 子弹类型分配

如果对局配置指定弹序列，则直接使用配置。

如果未指定弹序列，引擎默认策略为：**每发子弹独立随机**，50% 概率为 LIVE（实弹），50% 概率为 BLANK（空包）。

> 保证每轮**至少各有 1 发 LIVE 和 1 发 BLANK**。其余完全随机。

### 3.3 装弹流程

1. 如果对局配置指定弹序列，直接装入该序列
2. 如果未指定弹序列，生成 N 发子弹，向所有玩家展示 LIVE/BLANK 数量分布（但不显示顺序）
3. 未指定弹序列时，随机排列后装入弹匣

## 四、行动规则

### 4.1 开枪行动与回合切换

| 行动 | 子弹类型 | 伤害 | 回合切换 |
|------|---------|------|---------|
| **射自己** | BLANK | 0 | **不切换**，继续当前玩家回合 |
| **射自己** | LIVE | 1 HP（手锯→2） | **切换**到下一玩家 |
| **射对方** | BLANK | 0 | **切换**到下一玩家 |
| **射对方** | LIVE | 1 HP（手锯→2） | **切换**到下一玩家 |

### 4.2 啤酒退弹与回合消耗

- 非最后一发啤酒退弹：正常使用道具，**不消耗回合**
- **最后一发啤酒退弹：子弹打空本轮结束，且不保留回合**（按当前方向切换到下一玩家）

> 区别：射自己空弹即使轮次结束也保留回合，但啤酒退弹不行。

### 4.3 轮结束时的回合继承

轮次结束时，当前回合状态**继承到下一轮**：

| 最后一发是谁开的（或事件） | 子弹类型 | 下一轮第一回合 |
|---|---|---|
| 当前玩家射自己 | 空弹 | **该玩家继续** |
| 当前玩家射自己 | 实弹 | **按当前方向的下一玩家先手** |
| 当前玩家射对方 | 实弹/空弹 | **按当前方向的下一玩家先手** |
| 当前玩家啤酒退弹 | 最后一发 | **按当前方向的下一玩家先手** |

> 综合：射自己空弹且轮次正好结束 → 下轮还是你的回合。其他任何子弹耗尽的方式 → 按当前方向换人。

## 五、道具系统

### 5.1 道具列表（共 9 种）

| 编号 | 道具名 | 英文 | 效果                         |
|------|--------|------|----------------------------|
| ① | 干扰器 | Jammer | 选择一个其他存活玩家，该玩家的下一回合将被跳过。场上最多同时存在一个干扰器 |
| ② | 手锯 | Hand Saw | 增强当前子弹，如果是 LIVE，伤害翻倍（1→2 HP） |
| ③ | 放大镜 | Magnifying Glass | 查看**当前**子弹是 LIVE 还是 BLANK  |
| ④ | 啤酒 | Beer | 弹出当前子弹（揭示类型），弹匣推进一格        |
| ⑤ | 香烟 | Cigarette Pack | 回复 1 HP（不超过本场初始血量）         |
| ⑥ | 逆转器 | Inverter | **翻转**当前子弹类型（LIVE↔BLANK）   |
| ⑦ | 手机 | Burner Phone | 从当前弹之后的弹匣中**随机选一发**，揭示相对位置和类型    |
| ⑧ | 兴奋剂 | Adrenaline | **选择**对方一个道具（不能选兴奋剂），偷过来**立即使用** |
| ⑨ | 遥控器 | Remote | 反转当前回合方向。仅 3~4 人局发放 |

### 5.2 使用规则

- 一回合内可连续使用多个道具
- 使用道具**不消耗回合**（用完后仍可开枪），**最后一发啤酒退弹除外**
- 道具**一次性消耗**，使用后从背包移除

### 5.3 各道具详细逻辑

| 道具 | 逻辑                                                                |
|------|-------------------------------------------------------------------|
| 干扰器 | 创建一个场上干扰器，指定一名其他存活玩家。轮到目标玩家时自动跳过其回合并清除效果；若目标在触发前死亡，效果清除。**场上只能同时存在一个干扰器** |
| 手锯 | 设置 `self.hand_saw_active = True`。下一发子弹射出时若为 LIVE，伤害 ×2。              |
| 放大镜 | 记录当前子弹类型供展示                                                       |
| 啤酒 | 弹出当前子弹（揭示类型），`chamber_index += 1`。**最后一发时不保留回合**                  |
| 香烟 | `hp = min(hp + 1, max_hp)`                                        |
| 逆转器 | 翻转 `chambers[chamber_index]` 的 LIVE/BLANK。不推进弹匣。                  |
| 手机 | 多人模式下如果剩余子弹数 `<= 2`，道具失效并消耗；否则从 `chambers[chamber_index + 1:]` 中随机选一个位置，记录 `{offset, shell_type}` |
| 兴奋剂 | 玩家选择一名其他存活玩家，再从其 `items` 中选择一个非兴奋剂道具，**偷过来立即执行其效果**，不进入自己背包。场上玩家无可偷道具则无效 |
| 遥控器 | 仅 3~4 人局有效。执行 `turn_direction *= -1`，反转后续回合方向 |

### 5.4 道具发放

每轮道具数由对局配置决定。引擎默认策略：

```
每轮道具数 = min(floor(初始血量 / 2) + 轮数序号, 5)
```

| 初始血量 | 第 1 轮 | 第 2 轮 | 第 3 轮 |
|---------|---------|---------|---------|
| 3 | 1+1 = 2 | 1+2 = 3 | min(1+3, 5) = 4 |
| 4 | 2+1 = 3 | min(2+2, 5) = 4 | min(2+3, 5) = 5 |
| 5 | min(2+1, 5) = 3 | min(2+2, 5) = 4 | min(2+3, 5) = 5 |

从启用的道具池中完全随机抽取，同种可重复。

- 2 人局默认不发放遥控器（Remote）
- 对局配置可控制：每轮道具数、启用/禁用道具、单玩家同类道具上限、全场同类道具上限

### 5.5 道具栏上限

**每名玩家最多持有 8 个道具。**

该值由对局配置决定，8 是引擎默认值。

- 已有道具跨轮保留
- 新轮发放时叠加到存活玩家已有道具上
- **如果道具栏已满（已有 ≥8 个），无法再获得新道具**

## 六、游戏引擎文件结构

```
buckshot_roulette/
├── __init__.py
├── domain/
│   ├── models.py        # 游戏领域模型
│   └── engine.py        # 纯游戏引擎
└── rules.py             # 可选：规则常量与合法行动辅助
```

### domain/models.py 设计

```python
# 枚举
ShellType   = Enum: LIVE, BLANK
ItemType    = Enum: JAMMER, HAND_SAW, MAGNIFYING_GLASS, BEER, CIGARETTE_PACK, INVERTER, BURNER_PHONE, ADRENALINE, REMOTE
ActionType  = Enum: SHOOT_SELF, SHOOT_OPPONENT, USE_ITEM

# 玩家
Player:
  id: int                    # 固定座位编号，不随死亡或淘汰改变
  name: str
  hp: int                    # 当前血量
  max_hp: int                # 本场初始血量（回复上限）
  items: list[ItemType]      # 背包（上限 8）
  alive: bool                # 是否仍可行动/被选为合法目标
  hand_saw_active: bool      # 手锯激活标记

# 对局配置
MatchConfig:
  initial_hp_policy          # 初始血量策略：固定值/随机池
  shell_sequence_policy      # 弹序列策略：指定序列/随机生成
  items_per_reload_policy    # 每轮道具数策略：固定值/随机/默认公式
  enabled_items: set[ItemType]
  max_items_per_player: int
  item_player_limits: dict[ItemType, int]
  item_table_limits: dict[ItemType, int]

# 比赛状态（一场比赛内的状态）
MatchState:
  players: list[Player]      # 所有玩家座位，死亡玩家保留在列表中
  chambers: list[ShellType]  # 当前弹匣
  chamber_index: int         # 当前弹匣位置
  current_player_idx: int    # 当前行动玩家编号（对应 players 索引）
  turn_direction: int        # 当前回合方向：1 为正向，-1 为反向
  round_number: int          # 当前轮数（Reload Round）
  jammer_target: int | None  # 场上干扰器的目标玩家索引，无干扰器则为 None
  config: MatchConfig        # 本场比赛使用的对局配置
  last_action: dict          # 最近一次操作的结果信息
  match_over: bool           # 比赛是否结束
  winner: int | None         # 获胜者索引

# 整局游戏状态
GameState:
  match_results: list       # 每场比赛的胜者索引
  current_match: int        # 当前第几场（0-2）
  config: MatchConfig       # 整局使用的对局配置
  game_over: bool           # 整局是否结束
  final_winner: int | None  # 最终胜者
```

### domain/engine.py 核心函数

| 函数 | 职责 |
|------|------|
| `init_game(player_names, config=None)` | 创建新 GameState |
| `start_match(state)` | 开始一场新比赛（按配置设置血量） |
| `start_round(match_state)` | 开始新一轮装弹 |
| `generate_chambers(config)` | 按对局配置生成弹匣，展示分布后随机排列 |
| `distribute_items(match_state)` | 按对局配置发放道具 |
| `execute_action(match_state, action, target_idx=None)` | 执行行动 |
| `use_item(match_state, item_type, target_player_idx=None, target_item_idx=None)` | 使用道具 |
| `shoot_self(match_state)` | 射自己 |
| `shoot_opponent(match_state, target_idx)` | 射指定玩家 |
| `advance_chamber(match_state)` | 推进弹匣 |
| `next_turn(match_state)` | 按当前方向切换到下一玩家（跳过死亡/被干扰的） |
| `check_round_end(match_state)` | 检查子弹是否打空 |
| `check_match_end(match_state)` | 检查是否只剩一人 |
| `check_game_end(game_state)` | 检查是否打完 3 场 |

## 七、策略文档（strategy.md）

纯提示性策略：

```markdown
# 策略指南

## 基础原则
- 开枪前先分析概率：脑记已打出的子弹，推算剩余 LIVE/BLANK 分布

## 道具优先级
- 兴奋剂（场上存在强力道具）> 干扰器（控场，唯一）> 手锯（爆发）> 遥控器（3~4 人局改变节奏）
- 放大镜（当前）> 手机（未来）> 啤酒（消耗+信息）
- 经典 combo：放大镜+逆转器=必中；放大镜+逆转器+手锯=必重伤

## 信息利用
- 放大镜→当前子弹，手机→当前弹之后的随机一发，啤酒→消耗并揭示
- 结合已打出的子弹，可推断剩余弹匣的分布

## 跨轮节奏
- 最后一发射自己空弹 → 下轮还是自己回合
- 最后一发啤酒退弹 → 下轮按当前方向换人
- 遥控器会反转后续回合方向，3~4 人局可用来改变行动权压力
```
